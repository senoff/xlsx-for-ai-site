What you'll see (and what you won't)

You won't get an error. The file imports or opens “successfully,” but the result isn't what you expected — which is exactly why this slips through unnoticed.

Why it happens

A value in the wrong format, a blank where the system expected a number, or a column the software quietly ignored — small things that don't trip an error but change the outcome.

How to check

Upload the file above. The tool reads what's actually in it — not what you meant — so you can see the mismatch before it reaches your store or your report.

Let your agent clean duplicate rows before processing

When an agent pipeline receives a spreadsheet that may have repeated entries — a CRM export, a combined order log, a vendor list merged from two sources — it can run the same deduplication step automatically, with the same whitespace-aware comparison and the same before/after row count, before the data reaches any downstream system.

The tool this page runs

Your agent calls xlsx_data_clean in execute mode to remove duplicate rows — every change surfaces as a finding you can accept or reject before the file is written.

  • Options: dedup_compare_columns (which columns define a duplicate), dedup_ignore_case.
  • Run diagnose mode first to preview what would be removed.
