Let your agent screen files for macros automatically

If files arrive through an inbox, a shared drive, or an upload form, your agent can screen every one for macros and embedded links before a human ever opens it — flagging the risky ones and passing the clean ones through, no manual check required.

The tools this page runs

Your agent calls xlsx_macros — it reports whether a VBA macro project is present, its size, and the likely module names, and by policy never extracts or runs the macro code. Pair it with xlsx_external_links to list any embedded links to other workbooks.

  • Both take just the file — there are no options to set.
  • xlsx_macros returns presence, project size, and module-name candidates; xlsx_external_links returns each target path and whether it's absolute, a network share, or relative.
