Shopify silent import failure

Variants, sizes, or colors didn't come through after import — or they each became a separate product

You sell a t-shirt in Small, Medium, and Large. You imported your CSV and now Shopify has three separate products — one for each size — instead of one product with three size options. Or maybe even worse: only Small showed up and Medium and Large are just gone. Both problems come from the same place.

What your file needs to look like

For a product with three sizes, you need three rows, and every one of those rows needs the same Handle — that's what groups them into one product. The first row also needs the Title, the description, and the name of the option (like "Size"). The second and third rows only need the Handle (matching the first row), the option value (like "Medium" and "Large"), and variant-specific things like the price and Stock Keeping Unit (SKU) for that size. The Title and description only go on the first row.

If your spreadsheet has product information written out on every row — name and description on every size — that's the normal way to organize a product catalog, but it's not the shape Shopify wants. The conversion step restructures it.

The option columns — where sizes and colors actually go

The specific option values (the words "Small," "Medium," "Large," or "Blue," "Red," "Green") go in columns called Option1 Value, Option2 Value, and so on. The name of the option type (the word "Size" or "Color") goes in Option1 Name on the first row of the product. If your spreadsheet has size and color in separate columns with your own column names, those need to be mapped to the right Shopify columns — otherwise the options don't register.

How Shopify groups variants — what the Handle column actually does

In Shopify's Comma-Separated Values (CSV) format, a product with multiple variants (sizes, colors, materials — any options) is represented by multiple rows. The first row has the product name, description, images, and the first variant. Each additional row below it has the next variant — the next size, the next color. No name, no description on these extra rows. They're just variant data.

The thing that tells Shopify "these rows all belong to the same product" is the Handle column. Every row that belongs to the same product must have the same Handle value. If three rows all say blue-canvas-tote in the Handle column, Shopify groups them as one product with three variants. The first row becomes the product; the other rows become additional options on that product.

What went wrong in your file

The variant-grouping problem almost always comes from one of two things going wrong with the Handle column.

Scenario A: Each row has a unique Handle (or each row has its own product name in that column). This happens when someone builds their spreadsheet with one row per size — naturally, the way you'd think about it — and each row has its own product name or SKU in what they put in the Handle column. When Shopify sees five rows all with different Handles, it creates five separate products. Each one probably has its size baked into the name, like "T-Shirt Small" and "T-Shirt Medium," and each one only has a single variant.

Scenario B: Variant rows have a blank Handle. The first row has a Handle, but the rows for the additional sizes below it don't. Shopify can't attach a blank-Handle row to anything, so it either drops those rows silently or tries to create a new product with a blank handle (which also fails silently). Result: only the first variant appears, and the rest are just gone.
