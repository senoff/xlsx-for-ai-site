How it works

The plain-language pages on this site each do one job. This page is the technical backstop: it lists every file-based tool the server exposes, reads that tool’s own input definition, and builds the form for you. Upload your file, choose a tool, fill in any options, and run it — you get exactly what an AI agent would get calling the same tool directly.

A web visitor is an API user

Each tool here hits the same API route an MCP client uses. The page turns your choices into the request the tool expects and shows the result verbatim — text output inline, and a download when the tool produces a new file. There is no separate “web” behaviour to diverge from the API.

What’s covered

Any tool that reads a single uploaded workbook is available — reading, describing, converting, validating, cleaning, aggregating, sorting, and the rest. Tools that need two files, a saved session, or a secret token to post to an outside service aren’t here, because those don’t fit a “run this against my one file” form.

Your original file is safe

The tool reads your file in memory to run, then discards it. Your uploaded .xlsx is never changed and never stored — tools that transform your data hand back a new file to download, leaving your original untouched.

Want these inside your AI agent, on your own machine, with no upload at all? Install xlsx-for-ai and ask Claude or Cursor to run any of these tools on your spreadsheet.

Give your agent the whole toolbox

You just ran one operation on a file. Your AI agent can reach every one of these ~50 spreadsheet operations directly — reading, auditing, cleaning, converting, and writing .xlsx files.

With xlsx-for-ai installed, your agent picks the right operation for each task itself, chaining them without you driving each step.
